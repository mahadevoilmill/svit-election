## 🆕 New Features Added

### 1. **User Registration**
- Users can create their own accounts
- Page: `/register.html`
- Username must be at least 3 characters
- Password must be at least 6 characters
- Password strength indicator shows strength

### 2. **Password Reset**
- Users can reset their password
- Page: `/reset.html`
- No email verification needed (can be added later)
- Just provide username and new password

### 3. **Updated Login Page**
- Added "Create Account" link
- Added "Forgot Password" link
- Default admin account still works: `NEST` / `sardar123`

## ⚙️ Setup Required

### Step 1: Create Users Table in Supabase

Go to: https://app.supabase.com → Your Project → **SQL Editor** → **New Query**

Run this SQL:

```sql
CREATE TABLE IF NOT EXISTS users (
  id BIGINT PRIMARY KEY GENERATED BY DEFAULT AS IDENTITY,
  username VARCHAR(255) UNIQUE NOT NULL,
  password VARCHAR(255) NOT NULL,
  created_at TIMESTAMP DEFAULT NOW(),
  updated_at TIMESTAMP DEFAULT NOW()
);

-- Disable RLS for now (can be enabled with policies later)
ALTER TABLE users DISABLE ROW LEVEL SECURITY;

-- Insert default admin user
INSERT INTO users (username, password) 
VALUES ('NEST', 'sardar123')
ON CONFLICT DO NOTHING;

-- Grant permissions
GRANT ALL ON users TO public;
GRANT ALL ON users TO anon;
GRANT ALL ON users TO authenticated;
GRANT ALL ON users TO service_role;
```

### Step 2: Restart Server

```bash
pkill -f "node server.js"
sleep 2
npm start
```

### Step 3: Test

1. Go to **http://localhost:5000**
2. You can:
   - ✅ Login with default: `NEST` / `sardar123`
   - ✅ Click "Create Account" to register new user
   - ✅ Click "Forgot Password" to reset password
   - ✅ Use new account to login

## 🔒 Security Notes

1. Adding bcrypt for password hashing
2. Adding email verification for registration
3. Adding security questions for password reset
4. Adding rate limiting to prevent brute force
5. Adding session tokens/JWT

## 🐛 Troubleshooting

**"Username already exists"**
- That username is already registered
- Try a different username or reset password

**"User not found" (on password reset)**
- The username doesn't exist
- Create account first or use correct username

**Registration not working**
- Make sure users table is created in Supabase
- Check server logs for errors
- Verify Supabase credentials in .env
